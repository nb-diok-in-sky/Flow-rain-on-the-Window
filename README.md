# 雨滴滑落动画

雨滴滑落动画的实现需要通过物体对x、y轴进行随机偏移而得出来
基于这个问题 我们可以拆解出来
1 需要分x 和y方向进行不同的算法偏移
2 需要随机的参数来进行偏移
3隐藏的条件是 一般的雨滴可能存在最左边滑落到最右边这种夸张的效果 ，并且出现一定左右滑动的随机性，但在动画初期不选择考虑，
所以隐性限制是要将雨滴左右移动的范围控制在一个合适的范围内， 同时要保证一条雨滴从头滑倒尾的情况

自此，我们大概的分析清楚了需求，可以开始构建了

首先从雨滴开始建模，我们使用一个圆形作为雨滴的初步模型，先看图 这个图里面东西还是蛮多的

# Y方向动画
ok这里来看y方向上的移动
也就是雨滴下落的动画表现，我们预期希望这个雨滴他偶尔慢偶尔快的一个运动表现，符合现实中客观存在的雨滴下流场景 时快时慢的感觉
<img width="2499" height="657" alt="Y_Move" src="https://github.com/user-attachments/assets/28babb59-8d27-46d7-9a47-5f273af57429" />
这里的知识点主要在于函数的运用，原理是使用好你原来V方向的分量减去你算好的函数结果
剩下的就是准备一个函数了
这里可以去函数可视化网站自由发挥 要达到的节奏你自己喜欢就行
这里选用的是这样的：
使用V分量-0.29 * sin(sin(0.5*sin（time.y + Random * FlowIntensity）+(time.y * Random * FlowIntensity))+time.y * Random * FlowIntensity)
数学，很奇妙吧
这里使用负数乘以这个最后结尾的sin函数是因为
咱们要的函数效果是下坠 使用正数的话会呈现一个上升的状态，咱们不要那个
原理解析的话咱们不看那个函数 函数自身变化比较多，属于自己随便搓出来的
来看减法的逻辑
咱们V分量获得的是一各个从下到上0到1的值 接近1代表这个地方是可见，允许你展现你的内容，接近0或者成为0会被抹除
然后咱们加了这么一大堆的东西，并且把他变成了负号 你减了一个负数 那就是加了某种东西
以此为基础，咱们来看，首先那个随机数 是不断向下流动 并且在不断地变化中，我们原本平平的V分量会加上这一部分sin值 然后关键来了，这一个V分量平平从黑到白的这个区域在0-1的范围内
然后他这个一块区域会整体加上刚刚的那一大长串求出的sin值  然后就会因为点位上具体值的大小不同，出现白色的区域放大(都加了某个值）or缩小(都减了某个值) 然后因为那个sin函数给的比较神人，所以每个部位减去的东西也不同
然后它这一整个随机数又是通过floor方法来保持一整块区域都是这一个值的  这个Y方向的动画就是这样
一句话总结就是 所有相同的区块，减去了随机生成的数值，所以它所形成的图形就会该改变 

# X方向 

原理和Y方向的差不都 我们通过获取UV的U分量获得了一条条竖着的从左到右的从黑到白的渐变 并且我们的圆形会随着明暗交界线的地方进行运动
所以目标是操控这条明暗交界线 也就是控制值为0的部分，来进行规律的运动

<img width="1791" height="435" alt="X_Move" src="https://github.com/user-attachments/assets/7785017c-8eaf-43f8-97c2-9033d1cf075a" />

在X方向上的运动的的话 我希望他在某个固定的范围内进行循环运动（类似于sin函数会从0到1再到0的那种感觉）只需要他自然一些有一些卡顿
并且所有的雨滴不是同时进行左右摇摆 他们是分开的，以上述效果为目标，进行搭建动画
首先我们依旧是使用函数显性展现网站制作了一个合适的函数模型函数模型，具体是
clamp((cos（(UV.y+Random1）*Random2） * Power(sin（Random1+Random2）,6))/2.6   , 0.1, 1) +Random1 - 0.5
函数 很神奇吧 这样一来我们就获取了一个基本函数 其中，具体的参数都得自己根据实际情况进行调整

我们通过刚刚那个函数可以获得一个随机数然后把负责X轴移动的UV的U值拿来减去这个随机数 就是我们需要的X的运动路径

原理还和Y方向差不多 竖的条条们通过减去这个随机数的值进行使得自身的这个数值这样那个样的变化，因为数条右边的数值更大，所以一整条减去同一个数 右边的数通常会保持白色或者变换成灰色，这就达成了我们操控明暗交界线的效果
依旧是通过减法来控制黑白交接的位置
只不过从Y反向的从上往下看变成了从右往左看（因为此时白色一般会分布在边界的右边）并且由于那个神人函数 和随机数匹配，我们的运动就会呈现很大的随机性
